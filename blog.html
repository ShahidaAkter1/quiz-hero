1. Difference between map,foreach,find,filter?

map:

i) Return value:newArray will be created based on your callback function
ii) Original Array: not modified
iii)newArray is not created after the end of a method call
iv) map method is chainable

foreach:

i) Return value:undefine
ii) Original Array: not modified
iii)newArray is not created after the end of a method call
iv) foreach method is not chainable

filter:

i) filter() returns an array containing the element that satisfies the condition
ii) In filter(), whole array is iterated despite the fact that the element being searched for is present at the beginning.

find:

i) find() returns the element itself that satisfies the condition.
ii) find(), as soon as the element that satisfies the condition is found, it gets returned.

2. Difference between Local Storage, Session Storage and Cookies?

Local Storage:

i) Storages data in browser without any expiration date
ii) Saves data in key/ value pairs in the browser
iii)Data is not deleted when the browser is closed
iv) Data is available for future sessions

Session Storage:

i) Storages data in browser for a session
ii) Saves data in key/ value pairs in the browser
iii) Data is deleted when the browser is closed
iv)Data is not available for future sessions

Cookies::

i) Data stored in small text file
ii) Naturally sent by visited websites
iii) Sent to users computer through browser
iv) Is not a program, doesn’t perform any function
v) Stored in key-value pair

3. what is Event Loop? How does it work?

Event Loop:

The event loop is the secret behind JavaScript's asynchronous programming. JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading. Let's take a look at what happens on the back-end.
The Event Loop has one simple job — to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop.
